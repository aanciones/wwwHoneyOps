# Alerts Page

## Overview

The **Alerts Page** allows users to monitor, manage, and resolve security alerts generated by the system.  
Alerts group related activities and help identify suspicious or malicious behavior detected by honeypots.

This page is designed for **incident monitoring, alert triage, and response management**.

---

## Alerts List

The main table displays a list of alerts with the following information:

- **ID**: Unique identifier of the alert.
- **Start**: Date and time when the alert was first detected.
- **Honeypot**: Honeypot associated with the alert.
- **Origin IP**: Source IP address involved in the alert.
- **State**: Current status of the alert.
- **Last seen**: Most recent activity related to the alert.
- **Last notify**: Last notification sent for the alert.
- **Next notify**: Scheduled time for the next notification.
- **Action**: Access to manage and review alert details.

Alerts are displayed in chronological order and support pagination.

---

## Search and Filters

### Filter by Origin IP

A search field allows filtering alerts by **origin IP address**, helping focus on specific sources.

---

### Honeypot Filter

You can restrict the list to alerts generated by a specific **honeypot**.  
Selecting **All honeypots** displays alerts from all available honeypots.

---

### Status Filter

Alerts can be filtered by their current state:

- Active
- Resolved
- False positive
- Auto-closed
- (Additional legacy states may also appear)

---

### Page Size

The number of alerts displayed per page can be adjusted:

- 25
- 50
- 100
- 200

---

### Refresh

The **Refresh** button reloads the alerts list using the current filters to ensure up-to-date information.

---

## Alert States

Alerts can have different states that reflect their lifecycle:

- **Active / Open**: The alert requires attention.
- **Resolved**: The issue has been handled.
- **False Positive**: The alert was identified as non-threatening.
- **Auto-closed**: The system automatically closed the alert.
- **Acknowledged / Closed**: Informational or legacy states.

Visual indicators help quickly identify active alerts.

---

## Pagination

Navigation buttons allow browsing through alerts:

- **Previous**: View older alerts.
- **Next**: View newer alerts.

Buttons are disabled when no further pages are available.

---

## Managing an Alert

Clicking **Manage** on an alert opens a detailed dialog where users can review and update the alert.

---

### Alert Details

The management dialog provides:

#### General Information
- Current alert status
- Honeypot name
- Source IP address
- Time window analyzed

#### Quick Summary
A concise summary highlighting activity distribution (for example, attempts per port) to quickly understand the alert pattern.

---

### Notes

Users can add **optional notes** to an alert, such as:

- Investigation context
- Root cause analysis
- Decisions taken
- Next steps

Notes can be saved independently.

---

### Alert Actions

Available actions include:

- **Save notes**
- **Mark as resolved**
- **Mark as false positive**

Actions update the alert state immediately and are reflected in the alerts list.

---

## Error Handling

If alerts or alert details cannot be loaded, a message is displayed explaining the issue.  
Users can retry by refreshing the page.

---

## Typical Use Cases

- Reviewing newly generated alerts
- Investigating suspicious activity
- Tracking alert resolution status
- Documenting findings and decisions
- Managing false positives

---

## Notes

- Not all alerts require manual action.
- Alerts may update automatically based on system activity.
- Pagination and filters help manage large volumes of alerts efficiently.
