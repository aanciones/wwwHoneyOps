# General Navigation

This page explains how the HoneyOps console is organized and where to find the core workflows.

---

## What you can do in the console

- Deploy, enroll, and manage honeypots.
- Monitor alerts and triage security events.
- Define policies for services, whitelists, and notification behavior.
- Review activity across all honeypots.
- Manage licenses, users, and account settings.

![Console Navigation](navigation.png)

---

## Console layout

- **Left navigation** — primary sections; the current page is highlighted.
- **Quick onboarding** — a shortcut card to download the `.ova` and create a new honeypot.
- **Account block** — signed-in user info and sign out.
- **Top bar** — page title, date/time badge, **Download .ova**, and time zone selector (mobile includes a menu button).
- **Main workspace** — filters, tables, and detail views for the selected section.

---

## Console sections

- **Dashboard** — high-level status, trends, and quick visibility into activity.
- **Honey Pots** — create, enroll, monitor, and manage appliances.
- **Policies** — configure services, whitelists, and notification behavior.
- **Activity** — detailed event stream from honeypots.
- **Alerts** — view, triage, and resolve security alerts.
- **Licences** — license capacity and subscription actions.
- **Reports** — scheduled and on-demand exports.
- **Configuración** — syslog integration and user management.
- **Mi perfil** — billing details and security (password) settings.
