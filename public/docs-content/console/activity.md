# Activity Page

## Overview

The **Activity Page** provides a detailed view of all detected events and network activities generated by the system.  
It allows users to monitor, filter, search, and inspect activity records coming from different honeypots.

This page is designed for **operational monitoring, investigation, and analysis**.

---

## Main Features

### Activity List

The central table displays a list of activity records with the following information:

- **Time**: When the activity occurred.
- **Honeypot**: The honeypot where the event was detected.
- **Signature**: Description of the detected event or rule.
- **Source IP + Port**: Origin of the connection.
- **Destination IP + Port**: Target of the connection.
- **Protocol**: Network protocol involved.
- **Severity**: Importance level of the activity.
- **Alert ID**: Associated alert identifier (if any).
- **Action**: Access to detailed information.

Activities are displayed in chronological order and can be browsed using pagination controls.

---

## Search and Filtering

### Text Search

A search field allows filtering activities by:

- Signature text
- Source IP
- Destination IP
- Signature ID

Results update instantly as you type.

---

### Honeypot Filter

You can filter activities by selecting a specific **honeypot** from the dropdown menu.  
Selecting “All honeypots” displays activity from all available honeypots.

---

### Page Size

The number of activities shown per page can be adjusted:

- 25
- 50
- 100
- 200

---

## Refreshing Data

The **Refresh** button reloads the activity list using the current filters and settings, ensuring you are viewing the latest available data.

---

## Pagination and Ordering

The activity list supports cursor-based navigation:

- **First**: Jump to the newest available activities.
- **Previous**: Go to the previous page.
- **Next**: Go to the next page.
- **Last**: Jump to the oldest available activities.

The current page indicator shows:
- Page number
- Whether you are browsing from the newest or oldest records
- Number of items displayed

---

## Severity Levels

Each activity includes a **severity badge**:

- **Green**: Low severity
- **Amber**: Medium severity
- **Red**: High severity

These colors help quickly identify critical events.

---

## Viewing Activity Details

Clicking **View** on any activity opens a detailed modal window with extended information, including:

### General Information
- Timestamp
- Honeypot ID
- Flow ID
- Direction and interface

### Network Details
- Source and destination IPs and ports
- Protocol
- Packet source

### Detection Details
- Action taken
- Signature and category
- Signature ID and revision
- Severity level

### Traffic Metrics
- Packets sent to server and client
- Bytes sent to server and client

This detailed view is useful for investigation and forensic analysis.

---

## Error Handling

If data cannot be loaded, a warning message is displayed explaining the issue.  
Users can retry loading the data using the **Refresh** button.

---

## Typical Use Cases

- Monitoring recent honeypot activity
- Investigating suspicious network behavior
- Reviewing alerts and their underlying events
- Performing incident analysis and reporting

---

## Notes

- Data is loaded dynamically and reflects the current state of the system.
- Pagination ensures efficient navigation through large datasets.
- Not all activities are associated with alerts.
